## 一、核心法律风险地图

按风险等级排序。**🔴 高 = 可能导致下架/罚款；🟡 中 = 需要 mitigations；🟢 低 = 注意但不致命**。

|  #  | 风险类别                       | 等级 | 触发场景                                          | Mitigation                                                                                                                                                                                                                                                                          |
| :-: | :----------------------------- | :--: | :------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| L1  | **被认定为投资顾问业务**       |  🔴  | 任何形式的「建议买入/卖出」、个性化荐股、回报预测 | 见 §二 文案铁律；产品定位为「记账与可视化工具」                                                                                                                                                                                                                                     |
| L2  | **未持牌经营基金销售**         |  🔴  | 提供基金申购/赎回入口或导流                       | MVP 完全不做交易/导流；只读展示                                                                                                                                                                                                                                                     |
| L3  | **行情数据未经授权传播**       |  🔴  | 直接展示交易所原始行情数据未付费授权              | ⚠️ **2026-06-18 调研推翻旧口径**：Tushare / AKShare / 聚宽均为**非商用授权**，仅限 dev / 自用阶段；上架前必须按 [ADR 017](adr/017-data-governance-pipeline-and-source-licensing.md) 发版闸门替换为商用授权源（CN 公募基金历史净值需专项采购 Choice / iFinD 或等价）+ 标注延迟与来源 |
| L4  | **个人金融信息违规收集**       |  🔴  | 未经同意收集账户密码、银行卡信息                  | 永不收集；MVP 全手动录入持仓                                                                                                                                                                                                                                                        |
| L5  | **PIPL 个人信息处理违规**      |  🟡  | 跨境传输、未做隐私协议、未告知用途                | 见 §三                                                                                                                                                                                                                                                                              |
| L6  | **加密货币业务红线**           |  🟡  | 提供交易、撮合、ICO 信息、价格预测                | 仅作为资产类别记账；不展示交易入口；不包含国内交易所导流                                                                                                                                                                                                                            |
| L7  | **应用商店金融类目审核失败**   |  🟡  | 类目误选「金融」、含「股票推荐」等关键词          | 类目选「效率工具」；ASO 关键词不含敏感词                                                                                                                                                                                                                                            |
| L8  | **数据准确性导致的诉讼风险**   |  🟡  | 用户依赖错误数字做出决策造成损失                  | 全屏免责；价格延迟标识；用户协议明确「仅供参考」                                                                                                                                                                                                                                    |
| L9  | **AI 输出投资建议**            |  🟡  | LLM 在用户问询时输出明确的「应该买/卖」建议       | Prompt 强约束 + 输出后过滤敏感词 + 加 disclaimer                                                                                                                                                                                                                                    |
| L10 | **未备案 App 上架国内市场**    |  🟡  | 国内安卓商店强制要求 ICP 备案                     | Stage 4 启动 ICP 备案（30-60 天）                                                                                                                                                                                                                                                   |
| L11 | **软件著作权未登记**           |  🟢  | 影响国内安卓上架与维权能力                        | Stage 4 申请软著（30 天，¥300-1000）                                                                                                                                                                                                                                                |
| L12 | **第三方库的开源协议合规**     |  🟢  | GPL/AGPL 库被引入闭源商业产品                     | 用 `license-checker` 扫描；MVP 阶段坚守 MIT/Apache/BSD                                                                                                                                                                                                                              |
| L13 | **未成年人保护**               |  🟢  | 注册无年龄验证                                    | 用户协议中限定 18+；不主动推送                                                                                                                                                                                                                                                      |
| L14 | **税务合规（个人开发者收入）** |  🟢  | 海外订阅收入未申报                                | 自行咨询税务师；Stage 4 公司化后规范处理                                                                                                                                                                                                                                            |

## 二、对外文案合规清单

> 这是 **每一行 UI 文案、每一段官网描述、每一句 App Store 介绍** 都要遵守的铁律。

### 2.1 永不出现的词（立即触发整改）

| ❌ 禁用             | ✅ 改为                   |
| :------------------ | :------------------------ |
| 建议买入 / 应当买入 | 达到目标配置需要买入 X 股 |
| 建议卖出 / 应当卖出 | 达到目标配置需要卖出 X 股 |
| 投资建议 / 投资顾问 | 投资追踪 / 资产管理工具   |
| 推荐组合 / 优选组合 | 你的组合 / 用户自定义组合 |
| 必涨 / 大概率上涨   | (永远不预测涨跌)          |
| 收益保障 / 稳健收益 | 历史表现 / 历史收益       |
| 财富管理 / 智能投顾 | 资产追踪 / 资产可视化     |
| 一键交易 / 跟投     | (不应出现)                |
| 跑赢大盘 / 战胜市场 | 与基准比较 / 历史对比     |
| 选股策略 / 择时策略 | (不应出现)                |

### 2.2 必须出现的免责（在以下位置）

- **App 启动后首屏 onboarding 第二步**：「本工具仅用于个人投资追踪与可视化，不构成投资建议」
- **任何展示价格的位置（持仓表/详情页/图表）**：「数据仅供参考，可能延迟，最终以交易所/基金公司发布为准」
- **再平衡视图**：「本计算基于你设定的目标配置，不构成投资建议」
- **AI 对话窗口**：「以下分析为基于历史数据的统计描述，不构成投资建议；任何投资决策请独立判断」
- **设置 → 关于**：完整版产品边界声明
- **官网首页 footer + App Store 描述结尾**：免责声明

### 2.3 关于 AI 输出的额外约束

```
System prompt 必含：
1. 你是一个资产数据描述助手，不是投资顾问
2. 不输出「应该」「建议」「必须」等指示性动词
3. 不预测未来涨跌
4. 任何回答末尾必须附「以上分析仅供参考，不构成投资建议」
5. 用户问「该不该买」时，回答「这取决于你的目标配置和风险偏好；从你的当前组合看，[资产]占比为 X%，目标为 Y%」

输出后过滤：
- 检测到「建议买/卖」「应该买/卖」「会涨/跌」→ 重新生成或拒答
```

## 三、数据合规（PIPL / DSL）

### 3.1 PIPL（个人信息保护法）核心要求

| 要求                            | MVP 落地方式                                                    |
| :------------------------------ | :-------------------------------------------------------------- |
| **告知同意**                    | 注册时弹隐私政策，明确收集项 + 用途                             |
| **最小必要**                    | 注册仅需邮箱；不要手机号/身份证                                 |
| **单独同意（敏感个人信息）**    | 持仓金额属于敏感金融信息 → 单独勾选确认                         |
| **撤回同意**                    | 设置中提供「导出我的全部数据」+「删除账号」按钮                 |
| **跨境传输**                    | 数据存 Supabase 新加坡区 → 触发跨境传输条款；需在隐私协议中明示 |
| **个人信息保护影响评估（PIA）** | V1.0 上线前完成（律师协助起草模板）                             |

### 3.2 DSL（数据安全法）

- 持仓数据不属于「重要数据」，但累计大量用户后达到一定规模需关注
- MVP 阶段无须特别处理；Stage 4 律师 review 时确认

### 3.3 跨境传输合规（关键）

> 因 Supabase 在新加坡，所有用户数据均涉及跨境传输。

**MVP 阶段做法**（最小合规）：

- 隐私政策中**明确告知**「数据存储在新加坡 AWS 节点」
- 用户**明确勾选同意**跨境传输
- 不存储手机号、身份证、银行卡等高敏字段

**V1.0 阶段做法**（正式合规）：

- 完成 PIPL 第 38 条要求的「个人信息出境标准合同」备案
- 或转用国内云服务商（阿里云/腾讯云 + 自建 Supabase）

### 3.4 隐私协议必含条款

- [ ] 收集哪些信息（明确列举）
- [ ] 收集目的（每项一一对应）
- [ ] 存储位置与期限
- [ ] 第三方共享情况（Sentry / PostHog / 数据源服务商）
- [ ] 跨境传输声明
- [ ] 用户权利（访问/更正/删除/导出/撤回）
- [ ] 联系方式（邮箱必填）
- [ ] 未成年人保护条款
- [ ] 协议变更通知机制

> **不要随便用网上模板**。Stage 4 必须律师起草或 review。

## 四、应用商店上架要点

### 4.1 App Store（中国区）

| 要点         | 说明                                                           |
| :----------- | :------------------------------------------------------------- |
| **类目**     | 选「效率」（Productivity）或「财务」(Finance) 中**非交易子类** |
| **不选**     | 「投资」、「金融服务」（这些类目需要金融牌照）                 |
| **关键词**   | 不含「股票推荐」「基金推荐」「炒股」「投资顾问」等             |
| **截图**     | 不展示具体收益数字（避免暗示回报承诺）                         |
| **描述**     | 必含「本应用不提供投资建议」                                   |
| **审核重点** | 4.0 设计、5.0 法律（**金融监管 § 5.2.1**）                     |

### 4.2 Google Play

- 类目选「Finance」下的「Personal finance」
- 不需要中国监管资质
- 隐私政策必须有公开 URL

### 4.3 国内安卓商店

| 商店   | 要求                                                   |
| :----- | :----------------------------------------------------- |
| 华为   | ICP 备案 + 软著；金融类需金融牌照 → 类目选「实用工具」 |
| 小米   | 同上                                                   |
| OPPO   | 同上                                                   |
| vivo   | 同上                                                   |
| 应用宝 | 同上                                                   |

> **策略**：所有国内商店都按「记账/效率工具」类目报。一旦被分类为金融类，直接拒审无解。

### 4.4 Apple StoreKit / Google Play Billing 订阅合规

- 订阅价格、自动续费、取消方式必须在 paywall 页**显著告知**
- 苹果要求 paywall 链接到「使用条款」与「隐私政策」
- 国内安卓侧如做支付宝/微信支付，需走商户号（必须公司主体）

## 五、合规节奏与产品阶段对齐

```
Stage 0 ─── Stage 1 ──── Stage 2 ──── Stage 3 ──── Stage 4
预备       MVP-0      MVP-1自用      MVP-2闭测     V1.0公开
                                     ↓             ↓
                              [产品边界写文档]    [启动正式合规]
                              [文案铁律入 CI]    [律师初步咨询]
                                                  [ICP 备案]
                                                  [软著]
                                                  [PIA 评估]
                                                  [隐私协议正式版]
                                                  [应用商店材料]
```

**关键洞察**：

- Stage 0-3 几乎不需要正式合规投入（产品自用 + 闭门测试 + 不收费）
- 一旦决定 V1.0 公开（接受陌生用户 + 收费），合规成本一次性集中产生
- 这是**正确的省钱姿势**：不到必要时刻不烧合规预算

## 六、何时必须找正式律师

> 以下情况**必须**聘请有金融科技/TMT 方向经验的执业律师，不可只靠 AI / 模板：

1. ✅ V1.0 公开发布前（隐私协议 / 用户协议正式版）
2. ✅ 接入支付（订阅、内购）前
3. ✅ 接入 AI 对话功能后（输出可能涉及投资意见）
4. ✅ 出现第一个用户投诉/纠纷
5. ✅ 单月活跃用户 > 10,000（监管视野阈值）
6. ✅ 接入 A 股 / 基金数据源**之前**的资质审查（Tushare Pro 商业使用条款）
7. ✅ 注册公司主体的同时
8. ✅ 与任何金融机构（券商/基金公司）合作前

### 律师筛选建议

- 关键词：金融科技、TMT、互联网合规、PIPL
- 渠道：在创/天同/方达 等综合所；垂直所如「大成」
- 报价：初次咨询 ¥1,000-3,000；隐私协议起草 ¥3,000-8,000；完整 V1.0 合规审查 ¥10,000-30,000
- 谨慎选择：不熟悉互联网产品的传统律师容易过度保守

## 七、自查 checklist

> 任何 PR / 任何上线前都过一遍。

### 文案

- [ ] 全文搜索代码库 + 设计稿，无「建议买/卖」「应该买/卖」「推荐」
- [ ] 全文搜索，无「保障收益」「必涨」「跑赢」
- [ ] 所有价格展示位置含「仅供参考，可能延迟」
- [ ] 启动 onboarding 含产品边界声明
- [ ] 用户协议链接、隐私政策链接在 Settings 可见

### 数据

- [ ] 不收集手机号、身份证、银行卡
- [ ] 注册仅邮箱
- [ ] 持仓金额作为敏感信息单独同意
- [ ] 数据存储位置已在隐私政策声明
- [ ] 提供导出/删除入口

### 应用商店

- [ ] 类目选「效率/财务（非交易）」
- [ ] 关键词不含敏感词
- [ ] 截图无收益承诺
- [ ] 描述含免责声明

### 第三方

- [ ] 行情数据源全部为**商用授权**源（Tushare / AKShare / 聚宽为非商用授权，仅限 dev；上架前按 [ADR 017 发版闸门](adr/017-data-governance-pipeline-and-source-licensing.md) 清零）
- [ ] 数据源 token 全部服务端化，客户端 bundle 无任何数据源密钥（ADR 017 / 风险登记册 R7）
- [ ] 开源协议无 GPL/AGPL（用 license-checker 扫）
- [ ] 隐私政策列出所有第三方 SDK

### AI（如已接入）

- [ ] System prompt 含投资建议禁令
- [ ] 输出过滤已实现
- [ ] 每条回答末尾自动附 disclaimer

## 一、产品定位与一句话价值

> **产品愿景**：为中国全球资产配置者打造的智能投资追踪与再平衡助手 —— 把碎片化的多平台持仓变成清晰的全景图，把模糊的"该买点什么"变成具体的再平衡方案。

**目标用户**（按优先级）：
1. **核心人群**：在 A股 / 港股 / 美股 / 公募基金 / 加密货币 中持有 ≥3 类资产、跨 ≥2 个平台的中国投资者
2. **次级人群**：QDII 基金重仓、希望复盘配置比例的稳健型投资者
3. **延伸人群**：刚开始多元化配置的年轻投资者

**资产范围（终局即 MVP 终态）**：
- ✅ A 股、港股、美股（含 ETF）
- ✅ 公募基金（手动输入 + 净值查询）
- ✅ 加密货币
- ❌ 不做：私募基金、银行理财、可转债、外汇、大宗商品、NFT、房产估值

## 六、原版遗漏的关键议题

> 这一节是修订版的核心增量，原 `analysis_results.md` 完全没有覆盖。

### 6.1 收益率算法选择：TWR vs MWR

- **TWR（时间加权收益率）**：消除资金注入/取出影响，**衡量"基金经理"水平**，业内标准
- **MWR / IRR（资金加权收益率）**：反映**实际钱袋子的回报**，对个人投资者更有体感
- **决策**：MVP 实现 TWR + 简单累计收益率；MWR/IRR 在 V1.0 加入。两者都展示给用户，并加 tooltip 说明差异。

### 6.2 货币基准与汇率

- 用户必须能选择「报告货币」（默认 CNY，可切 USD / HKD）
- 持仓原始币种必须保留，**不能在录入时就转换**
- 汇率源 day1 接入（exchangerate.host 免费 / OpenExchangeRates 付费）
- 历史汇率与当日汇率分离存储（计算历史收益率必须用历史汇率）

### 6.3 公募基金的特殊性（原版只字未提）

- 净值是**T+1 单点**，没有实时价
- 申购/赎回有费率（前端费、后端费、持有期费），但 MVP **不计算**，仅以"成本价 + 持仓份额"为口径
- QDII 基金有**额度限购、外汇折算延迟**，UI 需有"数据可能延迟"标识
- 货币基金按"7日年化 / 万份收益"展示，与权益基金口径不同 → MVP 阶段统一按"净值"处理，不做特殊化

### 6.4 交易日历 / 时区

- A股、港股、美股开盘时间不同；节假日不同；夏令时影响美股
- 价格刷新策略要按市场分别处理（开盘中实时 / 闭市后冻结到最新收盘价）
- 后端用 UTC，UI 按用户时区展示

### 6.5 税务计算（V1.0 才做）

- A股：免税
- 港股通：20% 红利税（H 股 10%）
- 美股：30% 股息预扣（部分券商办理 W-8BEN 后 10%）
- 基金：分红再投免税、现金分红可能涉税
- MVP 阶段**只展示税前收益**，明确告知；税务在 V1.0 加可选模块

### 6.6 数据准确性的法律边界

- 任何价格延迟可能误导决策；需要在所有展示价格的位置显示**"仅供参考，可能延迟"**
- 任何"再平衡建议"的措辞要严格避免使用"应该买入"、"建议卖出"，改为"达到目标配置需要的份额变化为 X"
- 详见 `legal-risk-map.md`

### 6.7 端到端加密的真实代价

- E2EE 意味着服务端不能搜索、不能聚合、不能服务端计算 P/L
- MVP 推荐：**TLS + 静态加密（Supabase Vault）+ 敏感字段在前端额外加密**，而非真正的 E2EE
- 真 E2EE 推迟到 V1.5+，作为 Pro+ 卖点

## 七、产品边界声明

> 这一节将作为产品任何对外文案的合规基线。

**本产品是**：
- ✅ 个人投资记账与可视化工具
- ✅ 基于用户自行录入数据的统计与分析展示
- ✅ 帮助用户达成自己设定目标配置的计算工具

**本产品不是**：
- ❌ 投资顾问 / 投资建议提供方
- ❌ 交易撮合 / 经纪商
- ❌ 基金销售平台
- ❌ 行情数据正式发布机构

**对外文案铁律**（在所有 UI、官网、应用商店描述中执行）：
1. 永不出现"建议买入 / 卖出"
2. 永不出现"将获得 X% 收益"等承诺性表述
3. 价格、净值、再平衡数字必须配延迟与免责说明
4. 用"达到 / 偏离"目标配置，不用"该 / 应该"

# Sheet & Overlay 设计系统规范

> 本规范适用于移动端产品中所有 Sheet（半屏/全屏）、Overlay、Dialog 及相关模态层的设计与开发决策。  
> 目标读者：设计师、前端/移动端工程师、AI 辅助开发工具。  
> 最后更新：2026-05

---

## 一、核心原则

本规范基于一条统一的空间语义逻辑：

- **左侧 = 导航离开**（轻量、可逆的动作：后退、关闭 sheet）
- **右侧 = 操作执行**（需要用户明确意图的动作：确认、保存、扩展功能）

这条逻辑在不同场景下的具体表现会有差异（见下文各章节），但底层语义始终一致。

---

## 二、组件分类与选型

### 2.1 决策树

遇到需要弹出内容的场景时，按以下顺序判断：

1. **只需要告知结果、无需用户操作** → Toast / Snackbar
2. **需要用户做简短的二选一决策**（确认/取消、删除/保留） → Dialog（Alert）
3. **内容较多、有列表/滚动/表单/多步操作** → Sheet
4. **独立完整页面、不属于主导航栈** → 全屏 Overlay
5. **线性多步流程、属于主导航栈** → 全屏 Page（Push）

### 2.2 各组件定义

| 组件                  | 呈现方式                     | 入场动画   | 导航模型                     | 典型场景                   |
| --------------------- | ---------------------------- | ---------- | ---------------------------- | -------------------------- |
| **Toast / Snackbar**  | 顶部或底部轻提示             | 淡入/滑入  | 自动消失，无需用户操作       | 操作成功、网络状态         |
| **Dialog（Alert）**   | 居中浮层                     | 淡入+缩放  | 页面内按钮关闭，不可手势关闭 | 确认删除、权限请求         |
| **Sheet（半屏）**     | 底部弹出，露出遮罩层         | 从底部滑入 | 手势/遮罩/按钮关闭           | 筛选、选择、分享、快捷操作 |
| **Sheet（全屏）**     | 底部弹出至全屏，保留顶部圆角 | 从底部滑入 | Close（X）关闭               | 详情预览、编辑器           |
| **全屏 Overlay**      | 覆盖整屏                     | 从底部滑入 | Close（X）关闭               | 功能介绍、营销页、价格对比 |
| **全屏 Page（Push）** | 覆盖整屏                     | 从右侧推入 | Back（←）返回                | 导航栈中的常规页面         |

### 2.3 区分 Overlay 与 Push Page 的标准

判断标准不是视觉呈现，而是导航模型：

- **Overlay**：不在导航栈中，关闭后回到调起位置的完全相同状态。入场动画从底部滑入。使用 Close（X）。
- **Push Page**：在导航栈中，用户是通过路径"走到"此页面的。入场动画从右侧推入。使用 Back（←）。

规则简化：**从底部弹出 = Close，从右侧推入 = Back。** 动画方向即导航模型的视觉暗示。

## 三、Sheet 导航栏规范

### 3.1 半屏 Sheet

#### 默认结构

```
┌─────────────────────────────┐
│         ── (Grabber) ──     │  ← 可选，见 3.4
│  [Close/Back]   Title   [Action] │
│                             │
│         内容区域             │
└─────────────────────────────┘
```

- **左侧**：Close（X）或 Back（←），统一承载"离开当前上下文"的语义。
- **右侧**：操作按钮（Done、Save、筛选确认、更多菜单等），可为空。
- **标题**：居中，简洁描述 sheet 内容。可省略（如内容本身已足够清晰）。

#### 仅手势关闭的轻量 Sheet

当同时满足以下条件时，可省略显式 Close 按钮：

- Sheet 为固定尺寸（无多锚点拖拽）
- 内容为只读或简单选择（无用户输入/编辑态）
- 误关后可轻易重新唤起，无数据丢失风险

此时关闭方式为：Grabber 下滑 + 点击遮罩层。

典型场景：排序、筛选条件、分享面板、快捷操作菜单。

#### Back + Close 共存（Sheet 内多步流程）

当 sheet 内部有多步流程（如设置向导、多步选择）时：

```
┌─────────────────────────────┐
│       ── (Grabber) ──       │
│  [←] [X]    Step Title  [Action] │
│                             │
│         内容区域             │
└─────────────────────────────┘
```

- Back（←）在最左侧，Close（X）紧邻其右，两者均在左侧区域。
- 通过图标差异区分（← vs X），必要时加大间距或使用文字 label 降低误触风险。

### 3.2 全屏 Sheet / 全屏 Overlay

```
┌─────────────────────────────┐
│  [X]        Title     [Action] │
│                             │
│         内容区域             │
└─────────────────────────────┘
```

- 左侧：Close（X），因为全屏 sheet/overlay 不在导航栈中。
- 右侧：操作按钮（可选）。
- 建议保留顶部圆角，作为"这不是一个 push page"的视觉暗示。

### 3.3 全屏多步流程（KYC/KYB 等，Push Page）

```
┌─────────────────────────────┐
│  [←]        Title       [X] │
│  ████████░░░░ Step 2/5      │
│                             │
│         内容区域             │
│                             │
│  [       Continue →       ] │
│  (Save & Exit - 可选文字链)  │
└─────────────────────────────┘
```

- **左侧**：Back（←），单步回退，轻量可逆。
- **右侧**：Close（X），退出整个流程，属于重决策操作。
- **Close 必须附带确认弹窗**：提示用户进度可能丢失，并提供"保存草稿"选项（如产品支持）。
- **底部可补充 Save & Exit**：当某一步内容较长时，用户无需滚回顶部点 X。形式为文字链接，非主按钮。

**为什么 Close 在右侧（与 Sheet 规范不同）：** 全屏流程中的 close 意味着放弃整个流程进度，这是一个高权重操作，语义上更接近"操作"而非"导航"，因此归右侧。与核心原则一致：左侧 = 轻量可逆导航，右侧 = 需要明确意图的操作。

### 3.4 Grabber Bar 规范

**默认策略：半屏 Sheet 默认添加 Grabber。**

允许移除 Grabber 的条件（需同时满足）：

- Sheet 为固定尺寸（不支持多锚点拖拽切换）
- 页面内有明确的退出操作（按钮或 Close icon）

典型可移除场景：成功/失败反馈、确认提示、仅包含按钮的操作结果页。

**注意**：当 sheet 支持多尺寸锚点（如半屏 ↔ 全屏）时，Grabber 是必须的——它是用户发现"可以拉动调整大小"的唯一视觉线索。

## 四、Sheet 尺寸锚点

### 4.1 标准档位

| 档位       | 高度参考                 | 使用场景                     | 可拖拽       |
| ---------- | ------------------------ | ---------------------------- | ------------ |
| **Small**  | 内容自适应（≤ 屏幕 30%） | 简单选择、确认操作、少量选项 | 否，固定     |
| **Medium** | 约屏幕 50%               | 列表选择、筛选面板、轻量表单 | 可选         |
| **Large**  | 约屏幕 85%（露出遮罩层） | 详情预览、长表单、多内容展示 | 可选         |
| **Full**   | 全屏                     | 编辑器、独立功能页           | 否，固定全屏 |

### 4.2 多锚点行为

- 当 sheet 支持 Medium ↔ Large 切换时，必须显示 Grabber。
- 锚点切换应有弹性吸附效果（snap），避免停留在中间状态。
- 当内容量不足以支撑某个锚点时，不要启用该锚点。

## 五、Dismissibility（可关闭性）规则

### 5.1 分类

| 类型                    | 手势关闭 | 遮罩点击关闭 | 描述                               |
| ----------------------- | -------- | ------------ | ---------------------------------- |
| **Dismissible（默认）** | ✅       | ✅           | 大多数 sheet 的默认行为            |
| **Semi-dismissible**    | ✅       | ❌           | 需要一定程度的关注，但允许手势退出 |
| **Non-dismissible**     | ❌       | ❌           | 必须通过页面内按钮退出             |

### 5.2 Non-dismissible 的适用场景

- 用户必须明确同意/拒绝的内容（条款确认、权限请求）
- 不可跳过的关键步骤（强制更新提示、首次引导）
- 包含未保存用户输入且无自动保存的表单

**注意**：Non-dismissible 的 sheet 必须至少提供一个关闭路径（如"同意"+"拒绝"，或"稍后再说"），绝不能出现用户无法退出的死锁状态。

## 六、遮罩层（Scrim）规范

### 6.1 视觉

- 颜色：黑色，透明度 Light Mode 下 40%，Dark Mode 下 60%（可根据品牌调整，但需全局统一）。
- 全局使用同一遮罩层样式，不因 sheet 类型不同而变化。

### 6.2 交互

- **Dismissible sheet**：点击遮罩层关闭 sheet。
- **Non-dismissible sheet**：点击遮罩层无反应。可选择添加轻微抖动动效提示用户需要在 sheet 内操作。

---

## 七、Sheet 嵌套与堆叠

### 7.1 规则

- **最大嵌套层级：2 层。** 即允许从一个 sheet 中打开另一个 sheet 或 dialog，但不允许第三层。
- 如果业务流程需要 3 层以上，应将流程重构为全屏页面。
- **Sheet → Dialog**：允许（如在 sheet 内触发确认弹窗）。
- **Sheet → Sheet**：允许但需谨慎。第二层 sheet 应为轻量操作（如日期选择器、子项选择），不应为复杂流程。
- **Dialog → Sheet**：不允许。Dialog 是最高优先级的模态层。

### 7.2 视觉处理

- 第二层 sheet 弹出时，第一层 sheet 应有明显的层级退后效果（缩小+降低亮度），让用户感知到层级关系。
- 关闭第二层后，第一层恢复原始状态。

---

## 八、Sheet 内导航（多步流程）

### 8.1 转场方式

Sheet 内部多步流程的页面切换，统一使用**左右推入动画**（与 navigation stack 一致）。

不推荐使用原地淡入淡出替换——它会让用户丧失"我在第几步"的空间感知。

### 8.2 步骤指示器（Stepper）

- 当流程 ≥ 3 步时，建议在 header 下方添加步骤指示器（进度条或步骤点）。
- 步骤指示器为只读展示，不可点击跳转（避免跳过必填步骤）。

### 8.3 首步与末步的导航差异

- **第一步**：左侧显示 Close（X），因为此时 back 无意义（没有上一步）。
- **中间步骤**：左侧显示 Back（←），Close（X）紧邻其右或根据 3.1 的规则处理。
- **最后一步**：右侧的 Action 按钮应为最终确认（Submit、Done），左侧仍为 Back（←）。

## 九、键盘弹出行为

### 9.1 半屏 Sheet

- 键盘弹出时，sheet 整体上移，确保当前聚焦的输入框始终在键盘上方可见区域内。
- Sheet 不应因键盘弹出而变为全屏——保持其原有尺寸锚点，仅调整垂直位置。

### 9.2 全屏 Sheet / Page

- 键盘弹出时，页面内容区域滚动以露出输入框。
- 主操作按钮（如 Continue）应跟随键盘上移，或固定在键盘正上方（keyboard accessory view），确保用户无需收起键盘即可提交。

## 十、状态保持与恢复

### 10.1 规则

| 场景               | 误关后状态              | 处理方式                         |
| ------------------ | ----------------------- | -------------------------------- |
| 只读/选择类 sheet  | 不保持，重置            | 无需处理                         |
| 含用户输入的 sheet | 同一 session 内保持草稿 | 重新打开时恢复上次内容           |
| 长流程 sheet/page  | 提供保存草稿选项        | 关闭前确认弹窗，提供"保存并退出" |

### 10.2 关闭前确认的触发条件

当 sheet 内存在用户已修改但未保存的内容时，任何关闭动作（手势、遮罩点击、Close 按钮）都应触发确认弹窗，而非直接关闭。

## 十一、无障碍（Accessibility）

### 11.1 焦点管理

- Sheet 打开时，焦点应移至 sheet 内的第一个可交互元素。
- Sheet 关闭时，焦点应返回触发 sheet 的元素。
- Sheet 打开期间，底层内容应对辅助技术不可见（aria-hidden 或等效处理）。

### 11.2 手势替代

- 所有可通过手势关闭的 sheet，必须同时在 accessibility tree 中提供可聚焦的关闭操作。
- 仅靠 Grabber + 遮罩点击关闭的轻量 sheet，需确保屏幕阅读器能发现并激活关闭操作。

### 11.3 语义标注

- Sheet 应标注为 dialog 或 modal role。
- Close 按钮应有清晰的 accessible label（如"关闭"、"返回上一步"），不能仅依赖图标。

## 十二、动效规范

### 12.1 入场 / 退场

| 组件                 | 入场                    | 退场                  | 时长  |
| -------------------- | ----------------------- | --------------------- | ----- |
| 半屏 Sheet           | 从底部滑入 + 遮罩层淡入 | 向下滑出 + 遮罩层淡出 | 300ms |
| 全屏 Sheet / Overlay | 从底部滑入              | 向下滑出              | 350ms |
| Dialog               | 居中淡入 + 轻微缩放     | 淡出                  | 250ms |
| Toast                | 从顶部/底部滑入         | 滑出或淡出            | 200ms |

### 12.2 手势跟手

- 用户下滑手势应与 sheet 位置实时联动（跟手），而非触发后播放固定动画。
- 当下滑距离超过阈值（建议为 sheet 高度的 30%）时松手，触发关闭；未超过则回弹。

### 12.3 缓动函数

- 统一使用平台标准缓动曲线。iOS 建议 `UISpringTimingParameters` 或等效弹性曲线。
- 避免线性动画（感觉机械）和过度弹性动画（感觉不稳定）。

---

## 十三、Toast / Snackbar 与 Sheet 共存

- 当 sheet 已打开时，Toast 应显示在 sheet 之上（z-index 更高）。
- Toast 不应被 sheet 遮挡，也不应导致 sheet 位置偏移。
- 如果 Toast 的内容与当前 sheet 相关（如"保存成功"），Toast 应在 sheet 内部展示而非全局展示。

### 导航按钮位置速查

| 场景                         | 左侧                  | 右侧           |
| ---------------------------- | --------------------- | -------------- |
| 半屏 Sheet（单层）           | Close（X）            | Action（可选） |
| 半屏 Sheet（无按钮关闭）     | —                     | Action（可选） |
| Sheet 内多步流程（第一步）   | Close（X）            | Action（可选） |
| Sheet 内多步流程（中间步骤） | Back（←）+ Close（X） | Action（可选） |
| 全屏 Sheet / Overlay         | Close（X）            | Action（可选） |
| 全屏多步流程 Push Page       | Back（←）             | Close（X）     |
| 常规 Push Page               | Back（←）             | Action（可选） |

### Grabber 速查

| 条件                                    | 显示 Grabber              |
| --------------------------------------- | ------------------------- |
| 多锚点可拖拽 sheet                      | ✅ 必须                   |
| 固定尺寸 + 无显式关闭按钮               | ✅ 必须（唯一的关闭暗示） |
| 固定尺寸 + 有显式关闭按钮或 Action 按钮 | ⚪ 可省略                 |

### Dismissibility 速查

| 条件                          | 可手势关闭  | 可点遮罩关闭 |
| ----------------------------- | ----------- | ------------ |
| 只读/选择类内容               | ✅          | ✅           |
| 含用户输入（有自动保存）      | ✅          | ✅           |
| 含用户输入（无自动保存）      | ⚠️ 触发确认 | ⚠️ 触发确认  |
| 必须用户明确决策（条款/权限） | ❌          | ❌           |
